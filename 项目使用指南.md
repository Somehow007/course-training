## 系统概述

### 系统简介

培养方案管理系统是面向高校教务处、系主任等角色的信息化管理平台，提供培养方案的全生命周期管理功能，包括学院管理、专业管理、课程管理、培养方案管理以及版本控制等功能。

## 部署指南

### 一、数据库部署

#### 1. 创建数据库

```sql
CREATE DATABASE course_training 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

#### 2. 执行数据库脚本

```bash
mysql -u root -p course_training < course_training.sql
```

该脚本包含：
- 完整的表结构定义
- 初始化数据（学院、专业、课程、用户、字典等）
- 版本管理相关表和数据

#### 3. 验证数据库

```sql
-- 检查表是否创建成功
SHOW TABLES;

-- 检查初始数据
SELECT * FROM college;
SELECT * FROM sys_dict;
SELECT * FROM user;
```

#### 4. 创建数据库用户（生产环境推荐）

```sql
CREATE USER 'course_training'@'%' IDENTIFIED BY 'strong_password';
GRANT SELECT, INSERT, UPDATE, DELETE ON course_training.* TO 'course_training'@'%';
FLUSH PRIVILEGES;
```

### 二、后端部署

#### 1. 配置文件修改

编辑 `src/main/resources/application.yaml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://your-mysql-host:3306/course_training?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai
    username: your-username
    password: your-password
  data:
    redis:
      host: your-redis-host
      port: 6379
      password: your-redis-password  # 如果有密码

server:
  port: 8090
```

#### 2. 编译打包

```bash
cd /path/to/course-training
mvn clean package -DskipTests
```

生成的jar包位于 `target/tp-0.0.1-SNAPSHOT.jar`

#### 3. 启动服务

**方式一：直接运行**
```bash
java -jar target/tp-0.0.1-SNAPSHOT.jar
```

**方式二：后台运行**
```bash
nohup java -jar target/tp-0.0.1-SNAPSHOT.jar > app.log 2>&1 &
```

**方式三：使用systemd服务（推荐生产环境）**

创建服务文件 `/etc/systemd/system/course-training.service`：

```ini
[Unit]
Description=Course Training Application
After=syslog.target network.target

[Service]
Type=simple
User=appuser
ExecStart=/usr/bin/java -Xms512m -Xmx2g -jar /path/to/tp-0.0.1-SNAPSHOT.jar
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start course-training
sudo systemctl enable course-training
```

#### 4. 验证部署

访问API文档：`http://your-server:8090/doc.html`

### 三、前端部署

#### 1. 安装依赖

```bash
cd frontend
npm install
```

#### 2. 环境配置

编辑 `.env.production`：

```env
VITE_API_BASE_URL=http://your-server:8090
```

#### 3. 构建生产版本

```bash
npm run build
```

生成的文件位于 `frontend/dist/` 目录。

#### 4. Nginx配置

创建配置文件 `/etc/nginx/conf.d/course-training.conf`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /path/to/frontend/dist;
    index index.html;

    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://localhost:8090;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

重启Nginx：
```bash
sudo nginx -t
sudo systemctl restart nginx
```

### 四、Docker部署（可选）

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: course_training
    volumes:
      - mysql-data:/var/lib/mysql
      - ./course_training.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "3306:3306"

  redis:
    image: redis:6-alpine
    ports:
      - "6379:6379"

  backend:
    build: .
    ports:
      - "8090:8090"
    depends_on:
      - mysql
      - redis
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/course_training
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: rootpassword
      SPRING_REDIS_HOST: redis

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mysql-data:
```

启动服务：
```bash
docker-compose up -d
```

## 用户使用指南

### 登录系统

1. 打开浏览器，访问系统地址
2. 输入用户名和密码
3. 输入验证码
4. 点击"登录"按钮

### 功能模块

#### 1. 学院管理

**适用角色**：教务处

**功能**：
- 查看学院列表
- 新增学院
- 编辑学院信息
- 删除学院

**操作步骤**：
1. 点击左侧菜单"学院管理"
2. 在列表页面进行相应操作

#### 2. 专业管理

**适用角色**：教务处

**功能**：
- 查看专业列表
- 新增专业
- 编辑专业信息
- 删除专业
- 专业分类筛选（工学、理学、文科）

#### 3. 课程管理

**适用角色**：教务处

**功能**：
- 查看课程列表
- 新增课程
- 编辑课程信息
- 批量删除课程
- 按课程类型筛选

**课程类型说明**：
- 通识教育
- 学科专业基础
- 专业课程
- 实验实践环节
- 进阶研学

#### 4. 培养方案管理

**适用角色**：教务处、系主任

**功能**：
- 查看培养方案列表
- 创建培养方案
- 编辑培养方案详情
- 导入/导出Excel
- 版本管理

**导入Excel步骤**：
1. 进入培养方案详情页
2. 点击"导入"按钮
3. 下载模板（如需要）
4. 填写模板数据
5. 上传填写好的Excel文件

> **注意**：导入前请确保课程和学院已存在于系统中

#### 5. 版本管理

**适用角色**：教务处

**功能**：
- 查看版本历史
- 创建新版本
- 发布版本
- 版本对比
- 版本回滚
- 归档版本

**创建版本步骤**：
1. 点击左侧菜单"版本管理"
2. 点击"创建版本"按钮
3. 选择培养方案
4. 填写版本名称和变更说明
5. 点击"确定"

**版本对比步骤**：
1. 在版本列表中勾选两个版本
2. 点击"版本对比"按钮
3. 查看差异详情

**版本状态说明**：
| 状态 | 颜色 | 说明 |
|------|------|------|
| 草稿 | 灰色 | 版本创建后的初始状态 |
| 已发布 | 绿色 | 版本已正式发布 |
| 已归档 | 橙色 | 版本已归档 |
| 已回滚 | 红色 | 从该版本回滚后产生的状态 |

#### 6. 用户管理

**适用角色**：教务处

**功能**：
- 查看用户列表
- 新增用户
- 重置用户密码

## 常见问题

### Q1: 登录时提示"验证码错误"

**解决方案**：
- 检查验证码是否输入正确
- 刷新验证码后重新输入
- 清除浏览器缓存后重试

### Q2: 导入Excel时提示"课程不存在"

**解决方案**：
- 先在"课程管理"中添加缺失的课程
- 确保Excel中的课程名称与系统中的完全一致

### Q3: 版本对比时显示大量删除记录

**解决方案**：
- 检查版本快照数据是否完整
- 查看版本变更日志确认变更内容

### Q4: 系统响应缓慢

**解决方案**：
- 检查数据库连接池配置
- 检查Redis连接状态
- 查看服务器资源使用情况

### Q5: 无法访问API文档

**解决方案**：
- 确认后端服务已启动
- 检查端口是否被占用
- 访问地址：`http://server:8090/doc.html`
